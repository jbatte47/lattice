# Token usage rules

Lattice tokens are the source of truth for color, spacing, typography, and elevation. Use semantic
names that describe intent, not raw values. This keeps components consistent and keeps them easy to
retarget when themes change.

### Naming conventions

- Use the semantic hierarchy defined in Issue #9 (color, spacing, typography, elevation).
- Prefer intent-based tokens (for example `text-primary`, `bg-surface-default`).
- Avoid raw values in components (for example `#ffffff`, `16px`, `box-shadow: 0 2px 4px`).

### Dos and donts

Correct (semantic token):

```tsx
<div className="bg-surface-default text-primary">...</div>
```

Incorrect (raw value):

```tsx
<div style={{ background: '#ffffff', color: '#22252a' }}>...</div>
```

### Enforcement

- Review for raw values in CSS, inline styles, and component props.
- Prefer tokens in design reviews and code review checklists.
- If a token does not exist, add it before landing UI that depends on it.

### Mapping guide

Token groups map cleanly to CSS variables and Tailwind classes.
